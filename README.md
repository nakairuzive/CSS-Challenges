# CSS Challenges

If you’ve ever learned CSS, you know the feeling:
You understand the syntax, you know the properties… but when it’s time to *build something*, your mind suddenly goes blank.

That’s where **CSS challenges** come in.

This repository is inspired by the **CSS Challenges on Scrimba** and was created as a hands-on way to sharpen CSS skills by building small, focused UI components. Each challenge isolates a specific concept — animations, layouts, transitions, or visual tricks — so you can practice without being overwhelmed by a full project.

Think of this repo as a **CSS gym**: short reps, good form, steady progress.

##  Technologies

* HTML
* CSS

*(No frameworks, no libraries — just pure CSS)*

##  Challenges Included

This repository includes multiple bite-sized challenges, such as:

* Button hover effects
* Loaders and animations
* Expanding search bars
* Toggle switches
* Progress bars
* Flags built entirely with CSS
* Card flips and UI interactions

Each challenge focuses on **one main idea**, making it easier to understand *why* the CSS works instead of blindly copying code.

##  The Process

* Start by understanding **what the challenge is asking for visually**
* Break the design into smaller components (containers, elements, states)
* Write the HTML first — keep it clean and semantic
* Style progressively:

  * Layout first (flexbox/positioning)
  * Then colours, spacing, and sizing
  * Finally, animations and transitions
* Avoid overcomplicating things — most solutions are simpler than they look

##  What I Learned

* Small challenges are incredibly effective for mastering CSS fundamentals
* Animations make much more sense when you experiment instead of memorising syntax
* Naming classes properly saves *a lot* of confusion later
* You don’t need JavaScript to create interactive-looking components
* Consistency in spacing and sizing makes designs feel more “professional”

##  What Can Be Improved

* Better folder structure as the number of challenges grows
* Adding comments to explain *why* certain CSS choices were made
* Improving responsiveness for different screen sizes
* Creating variations of the same challenge to explore alternative solutions

##  How to Use This Repo

* Clone or fork the repository
* Open any challenge folder
* Open the `index.html` file in your browser
* Modify the CSS and experiment — break things on purpose

This repo is meant for **learning**, not perfection.

##  Final Thoughts

CSS can feel frustrating, but it becomes much more enjoyable when you treat it like a creative tool instead of a checklist of properties. These challenges helped me build confidence, develop intuition, and actually *enjoy* styling again.

If you’re learning CSS — or revisiting it — I hope this repo helps you the same way Scrimba’s challenges helped inspire it.
